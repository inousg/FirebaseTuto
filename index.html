Database

  • What is a database ?

    A database (DB) is a structured collection of data, accessible and usable via computer programs. It is always paired with a Database management system (DBMS).
    A DBMS is a software system that enables users to define, create, maintain and control access to the database. Its core functionality is the storage, retrieval and update of data.

  • How to create a database ? (step #1)

    First you have to choose a DBMS, like MySQL or MongoDB. In fact there are many ways to deal with a DB, so you need to select a DMBS accordingly.
    A relational database for example organizes its data through multiple tables (called relations) of columns and rows, with a unique key identifying each row. To use it you will need a relational database management system (RDBMS).

  • How to create a database ? (step #2)

    Once you select a DBMS, you have to learn a query language it features. SQL (Structured Query Language) the most famous one is often used to manage relational databases.
    From now on, you can administrate your database thanks to this language. It is possible to create some tables, update and delete them. You can also select particular data from your DB.

Firebase

  • What is Firebase ?

    Firebase is a mobile and web application development platform currently owned by Google. It helps developers to build, improve, and grow an application. This includes things like analytics, authentication, databases, configuration, file storage, push messaging, and the list goes on.
    You are going to use one of these services : Firebase Realtime Database.

  • Step #1 - Create a project

    Go to Firebase, click get started and create a new project. Enter the name of your project and do not enable Google analytics, you don't need it.
    Once the project created, click on it to discover the project dashboard. You must now add your application to the project. Choose a web app but do not set up Firebase Hosting. When you are finished, Firebase must have given you some scripts.

  • Step #2 - Recover the scripts

    Recover these scripts and add them at the bottom of your html page as asked. If you didn't copy them you can retrieve them in your project settings.

  • Step #3 - Create your database

    Now go to your project dashboard and click on Database on the navbar. Do not use Cloud Firestore but scroll down, create a "Realtime Database" and start in test mode so you can access your data from your local machine.
    Tadaaa ! You have created your database, try to modify the data manually.

  • A NoSQL Database

    Before we go further, you must know that Firebase does not use a relational database but a NoSQL Database (non relationnal database). It means it doesn't use tables like we have seen precedently.
    Firebase implements a document-oriented database, based on the JSON data structure.

  • JSON data-structure

    The firebase database is in fact a single JSON tree. You can read the doc about it here.
    JSON objects are written in key/value pairs. When you start adding data to your database it automatically creates a node in the existing JSON structure with an associated key.
